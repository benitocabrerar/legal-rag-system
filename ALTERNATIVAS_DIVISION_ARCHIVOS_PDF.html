Limitaciones Identificadas

⚠️ Problema Actual: Archivos PDF Grandes

  • Límite actual: 50 MB (frontend + backend)
  • Render Free Tier: 256 MB RAM, timeouts de 60 segundos
  • Caso problemático: PDF de 500 páginas (~100-150 MB)
  • Riesgos:
    • Out of Memory (OOM) durante procesamiento
    • Timeouts en extracción de texto con pdf.js-extract
    • Costos elevados en OpenAI embeddings (1 archivo = miles de chunks)
    • Lentitud en búsquedas semánticas

Procesamiento Actual

1. Carga de PDF
Multer/Multipart → Buffer (50 MB max)
↓
2. Extracción de Texto
pdf.js-extract → String completo
↓
3. Chunking
Divide en fragmentos de 1000 caracteres
↓
4. Embeddings
OpenAI API por cada chunk
↓
5. Almacenamiento
PostgreSQL (LegalDocumentChunk)

📋 Caso de Uso: PDF de 500 Páginas

Ejemplo: Código Civil del Ecuador

Escenario real: Subir el Código Civil completo (500+ páginas, ~120 MB)

Desafíos Actuales:

  • Archivo excede el límite de 50 MB
  • Procesamiento completo tomaría 5-10 minutos (timeout 60s)
  • Generaría ~1,200 chunks (500 páginas × 2.4 chunks/página promedio)
  • Costo OpenAI: $0.12 - $0.15 USD por documento (1,200 chunks × ~1,000 tokens × $0.0001)
  • Consumo RAM: 150-200 MB durante procesamiento (excede límite free tier)

Solución Propuesta:

Dividir en 5 archivos de 100 páginas cada uno

  • Cada archivo: ~24 MB (dentro del límite)
  • ~240 chunks por archivo (manejable)
  • Procesamiento paralelo o secuencial
  • Manejo de secuencia y relaciones entre archivos

Alternativa 1: División Física del PDF (Múltiples Archivos)

RECOMENDADA Implementación Media

Descripción

Dividir el PDF original en múltiples archivos más pequeños antes de procesarlos. Cada archivo se sube y procesa independientemente.

Código de Ejemplo - División con pdf-lib

import { PDFDocument } from 'pdf-lib';

async function splitPDF(pdfBuffer: Buffer, pagesPerPart: number = 100) {
  const pdfDoc = await PDFDocument.load(pdfBuffer);
  const totalPages = pdfDoc.getPageCount();
  const totalParts = Math.ceil(totalPages / pagesPerPart);
  const parts: Buffer[] = [];

  for (let i = 0; i < totalParts; i++) {
    const partDoc = await PDFDocument.create();
    const startPage = i * pagesPerPart;
    const endPage = Math.min((i + 1) * pagesPerPart, totalPages);

    // Copiar páginas
    const copiedPages = await partDoc.copyPages(
      pdfDoc,
      Array.from({ length: endPage - startPage }, (_, idx) => startPage + idx)
    );

    copiedPages.forEach(page => partDoc.addPage(page));

    const pdfBytes = await partDoc.save();
    parts.push(Buffer.from(pdfBytes));
  }

  return parts;
}

// Uso
const originalPDF = await readFile('codigo-civil-500-pages.pdf');
const parts = await splitPDF(originalPDF, 100); // 5 partes de 100 páginas

// Guardar cada parte
for (let i = 0; i < parts.length; i++) {
  await prisma.legalDocumentPart.create({
    data: {
      parentDocumentId: documentId,
      partNumber: i + 1,
      totalParts: parts.length,
      startPage: i * 100 + 1,
      endPage: Math.min((i + 1) * 100, 500),
      content: '', // Se llenará durante procesamiento
      fileSizeMB: parts[i].length / (1024 * 1024),
      isProcessed: false,
      processingStatus: 'pending'
    }
  });
}
✅ Ventajas
  • Cada parte permanece dentro del límite de 50 MB
  • Procesamiento paralelo posible (5 workers simultáneos)
  • Si falla una parte, las demás ya están procesadas
  • Mejor para memoria RAM (procesa 24 MB en vez de 120 MB)
  • Usuario puede descargar/ver partes individuales
  • Permite reintento granular por parte fallida
❌ Desventajas
  • Requiere librería adicional (pdf-lib)
  • Duplicación de almacenamiento (archivo original + partes)
  • Complejidad en búsqueda (debe buscar en todas las partes)
  • Referencias entre páginas pueden perderse
  • Requiere lógica adicional de unión para visualización completa

💡 Caso de Uso Óptimo

Documentos legales extensos que rara vez se consultan completos (códigos, compendios, jurisprudencias voluminosas). Ideal cuando se necesita:

  • Procesamiento robusto con reintentos granulares
  • Distribución de carga entre múltiples workers
  • Descarga selectiva (solo las secciones necesarias)

Código de Ejemplo - Procesamiento por Bloques

import { PDFExtract } from 'pdf.js-extract';

async function processLargePDFInBatches(
  pdfBuffer: Buffer,
  documentId: string,
  pagesPerBatch: number = 100
) {
  const pdfExtract = new PDFExtract();
  const data = await pdfExtract.extractBuffer(pdfBuffer);
  const totalPages = data.pages.length;

  // Inicializar progreso
  await prisma.legalDocument.update({
    where: { id: documentId },
    data: {
      totalPages,
      processingProgress: {
        totalChunks: 0,
        processedChunks: 0,
        pageRanges: []
      },
      processingStage: 'extracting_text'
    }
  });

  // Procesar en bloques de 100 páginas
  for (let i = 0; i < totalPages; i += pagesPerBatch) {
    const endPage = Math.min(i + pagesPerBatch, totalPages);
    const pageRange = { start: i + 1, end: endPage, status: 'processing' };

    // Extraer texto de este bloque
    const blockText = data.pages
      .slice(i, endPage)
      .map(p => p.content.map(c => c.str).join(' '))
      .join('\n\n');

    // Generar chunks para este bloque
    const chunks = chunkDocument(blockText, 1000);

    // Generar embeddings
    for (let j = 0; j < chunks.length; j++) {
      const embedding = await generateEmbedding(chunks[j]);

      await prisma.legalDocumentChunk.create({
        data: {
          legalDocumentId: documentId,
          content: chunks[j],
          chunkIndex: i * 2.4 + j, // Estimación
          embedding
        }
      });
    }

    // Actualizar progreso
    await updateProcessingProgress(documentId, pageRange, 'completed');
  }

  // Marcar como completado
  await prisma.legalDocument.update({
    where: { id: documentId },
    data: { processingStage: 'completed' }
  });
}
✅ Ventajas
  • No duplica archivos (un solo PDF)
  • Implementación más simple (sin división física)
  • No requiere librerías adicionales de PDF
  • Progreso visible en tiempo real
  • Puede reanudar desde el último bloque procesado
  • Búsquedas más simples (un solo documento)
❌ Desventajas
  • Todavía carga el archivo completo en memoria (120 MB)
  • Si falla, debe reprocesar desde el inicio del bloque actual
  • No permite procesamiento paralelo verdadero
  • Requiere manejo sofisticado de estado
  • Puede seguir causando OOM en Render Free Tier

💡 Caso de Uso Óptimo

Cuando ya tienes el archivo completo y solo necesitas mejorar la robustez del procesamiento. Ideal para:

  • Sistema con suficiente RAM (512 MB+)
  • Archivos de tamaño moderado (50-100 MB)
  • Cuando no puedes dividir físicamente el PDF

Alternativa 3: Streaming + Procesamiento en Chunks

Complejidad Alta Eficiencia Máxima

Descripción

Procesar el PDF página por página usando streams, sin cargar el archivo completo en memoria.

pdf-parse Node.js Streams pdf2json Transform Streams

Concepto con Streams

import { Readable } from 'stream';
import pdf from 'pdf-parse';

async function processLargePDFWithStreaming(
  fileStream: Readable,
  documentId: string
) {
  let pageBuffer = '';
  let pageCount = 0;
  let chunkBuffer: string[] = [];

  // Crear transform stream que procesa página por página
  fileStream
    .pipe(createPDFTransformStream())
    .on('data', async (page) => {
      pageCount++;
      pageBuffer += page.text + '\n\n';

      // Cuando acumulamos ~5 páginas, procesamos
      if (pageCount % 5 === 0) {
        const chunks = chunkDocument(pageBuffer, 1000);

        for (const chunk of chunks) {
          const embedding = await generateEmbedding(chunk);
          await prisma.legalDocumentChunk.create({
            data: {
              legalDocumentId: documentId,
              content: chunk,
              chunkIndex: chunkBuffer.length,
              embedding
            }
          });
          chunkBuffer.push(chunk);
        }

        pageBuffer = ''; // Limpiar buffer
      }
    })
    .on('end', async () => {
      // Procesar páginas restantes
      if (pageBuffer.length > 0) {
        // ... procesamiento final
      }

      console.log(`✅ Procesadas ${pageCount} páginas, ${chunkBuffer.length} chunks`);
    })
    .on('error', (error) => {
      console.error('❌ Error en streaming:', error);
    });
}
✅ Ventajas
  • Consumo mínimo de memoria (solo 5 páginas en buffer)
  • Puede procesar archivos de cualquier tamaño
  • Eficiente para el plan Free de Render
  • Procesamiento verdaderamente incremental
❌ Desventajas
  • Complejidad de implementación muy alta
  • Difícil manejo de errores y reintentos
  • No todas las librerías PDF soportan streaming
  • Debugging complicado
  • Puede ser más lento que procesamiento en batch

Alternativa 5: Storage Externo + Referencias

Cloud Storage Costos Adicionales

Descripción

Almacenar el PDF completo en S3/Cloudinary y solo guardar referencias + chunks procesados en PostgreSQL.

Flujo con S3

1. Usuario sube PDF grande (120 MB)
   ↓
2. Backend lo almacena en S3
   - URL: https://bucket.s3.amazonaws.com/legal-docs/codigo-civil.pdf
   ↓
3. Se crea registro en LegalDocument con:
   - storageUrl: URL de S3
   - storageProvider: "s3"
   ↓
4. Worker en background descarga de S3 en chunks
   - Procesa 10 MB a la vez
   - Genera embeddings
   - Guarda chunks en PostgreSQL
   ↓
5. Una vez procesado, puede eliminar de S3 (opcional)
   o mantener para descarga completa
✅ Ventajas
  • Sin límites de tamaño (S3 soporta hasta 5 TB)
  • No consume RAM del servidor durante upload
  • CDN para descargas rápidas
  • Backup automático del archivo original
❌ Desventajas
  • Costos adicionales ($0.023/GB/mes en S3)
  • Requiere configuración de AWS/Cloudinary
  • Dependencia de servicio externo
  • Latencia en descarga desde S3 para procesamiento

import { PDFDocument } from 'pdf-lib';

async function splitPDFInBrowser(file: File, pagesPerPart: number = 100) {
  // Leer archivo
  const arrayBuffer = await file.arrayBuffer();
  const pdfDoc = await PDFDocument.load(arrayBuffer);
  const totalPages = pdfDoc.getPageCount();

  const parts: { blob: Blob, partNumber: number, pages: string }[] = [];

  // Dividir
  for (let i = 0; i < totalPages; i += pagesPerPart) {
    const partDoc = await PDFDocument.create();
    const endPage = Math.min(i + pagesPerPart, totalPages);

    const copiedPages = await partDoc.copyPages(
      pdfDoc,
      Array.from({ length: endPage - i }, (_, idx) => i + idx)
    );

    copiedPages.forEach(page => partDoc.addPage(page));

    const pdfBytes = await partDoc.save();
    const blob = new Blob([pdfBytes], { type: 'application/pdf' });

    parts.push({
      blob,
      partNumber: parts.length + 1,
      pages: `${i + 1}-${endPage}`
    });
  }

  return parts;
}

async function uploadParts(parts: any[], documentMetadata: any) {
  for (const part of parts) {
    const formData = new FormData();

      method: 'POST',
      body: formData

Paso 4: Servicio de División de PDFs

// src/services/pdf-splitter.ts
import { PDFDocument } from 'pdf-lib';

export interface PDFPart {
  partNumber: number;
  totalParts: number;
  startPage: number;
  endPage: number;
  buffer: Buffer;
  sizeMB: number;
}

export class PDFSplitter {
  async split(pdfBuffer: Buffer, pagesPerPart: number = 100): Promise {
    const pdfDoc = await PDFDocument.load(pdfBuffer);
    const totalPages = pdfDoc.getPageCount();
    const totalParts = Math.ceil(totalPages / pagesPerPart);
    const parts: PDFPart[] = [];

    for (let i = 0; i < totalParts; i++) {
      const partDoc = await PDFDocument.create();
      const startPage = i * pagesPerPart;
      const endPage = Math.min((i + 1) * pagesPerPart, totalPages);

      // Copiar páginas
      const pageIndices = Array.from(
        { length: endPage - startPage },
        (_, idx) => startPage + idx
      );
      const copiedPages = await partDoc.copyPages(pdfDoc, pageIndices);

      copiedPages.forEach(page => partDoc.addPage(page));

      const pdfBytes = await partDoc.save();
      const buffer = Buffer.from(pdfBytes);

      parts.push({
        partNumber: i + 1,
        totalParts,
        startPage: startPage + 1,
        endPage,
        buffer,
        sizeMB: buffer.length / (1024 * 1024)
      });
    }

    return parts;
  }

  async merge(parts: Buffer[]): Promise {
    const mergedPdf = await PDFDocument.create();

    for (const partBuffer of parts) {
      const partDoc = await PDFDocument.load(partBuffer);
      const copiedPages = await mergedPdf.copyPages(
        partDoc,
        partDoc.getPageIndices()
      );
      copiedPages.forEach(page => mergedPdf.addPage(page));
    }

    const pdfBytes = await mergedPdf.save();
    return Buffer.from(pdfBytes);
  }
}

import { PDFSplitter } from '../services/pdf-splitter';
import { PDFExtract } from 'pdf.js-extract';

fastify.post('/legal-documents-v2/upload-large', {
  onRequest: [fastify.authenticate],
}, async (request: FastifyRequest, reply: FastifyReply) => {
  try {
    const user = request.user as any;

    if (user.role !== 'admin') {
      return reply.code(403).send({ error: 'Forbidden' });
    }

    if (request.isMultipart()) {
      const parts = request.parts();
      let fileBuffer: Buffer | null = null;
      let formData: any = {};

      for await (const part of parts) {
        if (part.type === 'file') {
          fileBuffer = await part.toBuffer();
        } else {
          formData[part.fieldname] = (part as any).value;
        }
      }

      if (!fileBuffer) {
        return reply.code(400).send({ error: 'No file provided' });
      }

      const fileSizeMB = fileBuffer.length / (1024 * 1024);

      // Si el archivo es mayor a 50 MB, dividirlo
      if (fileSizeMB > 50) {
        const splitter = new PDFSplitter();
        const pdfParts = await splitter.split(fileBuffer, 100);

        // Crear documento padre
        const parentDoc = await prisma.legalDocument.create({
          data: {
            normType: formData.normType,
            normTitle: formData.normTitle,
            legalHierarchy: formData.legalHierarchy,
            publicationType: formData.publicationType,
            publicationNumber: formData.publicationNumber,
            uploadedBy: user.id,
            content: '', // Se llenará con las partes
            metadata: {
              originalFileSizeMB: fileSizeMB,
              totalPages: pdfParts[0].totalParts * 100, // Aproximado
              dividedIntoParts: true
            }
          }
        });

        // Crear registros de partes
        for (const part of pdfParts) {
          await prisma.legalDocumentPart.create({
            data: {
              parentDocumentId: parentDoc.id,
              partNumber: part.partNumber,
              totalParts: part.totalParts,
              startPage: part.startPage,
              endPage: part.endPage,
              content: '', // Se llenará en background
              fileSizeMB: part.sizeMB,
              isProcessed: false,
              processingStatus: 'pending',
              metadata: { bufferLength: part.buffer.length }
            }
          });

          // Encolar job de procesamiento
          await documentQueue.add('process-document-part', {
            parentDocumentId: parentDoc.id,
            partNumber: part.partNumber,
            pdfBuffer: part.buffer.toString('base64')
          });
        }

        return reply.send({
          success: true,
          documentId: parentDoc.id,
          message: `Documento dividido en ${pdfParts.length} partes`,
          parts: pdfParts.length
        });
      } else {
        // Procesamiento normal para archivos pequeños
        // ... código existente ...
      }
    }
  } catch (error: any) {
    fastify.log.error(error);
    return reply.code(500).send({ error: error.message });
  }
});

Paso 6: Worker para Procesar Partes

// src/workers/document-part-processor.ts

import { Job } from 'bullmq';
import { PDFExtract } from 'pdf.js-extract';
import { prisma } from '../lib/prisma';
import { generateEmbedding, chunkDocument } from '../lib/utils';

export async function processDocumentPart(job: Job) {
  const { parentDocumentId, partNumber, pdfBuffer } = job.data;

  try {
    // Convertir base64 a buffer
    const buffer = Buffer.from(pdfBuffer, 'base64');

    // Extraer texto
    const pdfExtract = new PDFExtract();
    const data = await pdfExtract.extractBuffer(buffer);
    const text = data.pages
      .map(p => p.content.map(c => c.str).join(' '))
      .join('\n\n');

    // Actualizar parte con contenido
    await prisma.legalDocumentPart.update({
      where: {
        parentDocumentId_partNumber: {
          parentDocumentId,
          partNumber
        }
      },
      data: {
        content: text,
        processingStatus: 'processing'
      }
    });

    // Generar chunks
    const chunks = chunkDocument(text, 1000);

    // Generar embeddings y guardar chunks
    for (let i = 0; i < chunks.length; i++) {
      const embedding = await generateEmbedding(chunks[i]);

      await prisma.legalDocumentChunk.create({
        data: {
          legalDocumentId: parentDocumentId,
          content: chunks[i],
          chunkIndex: (partNumber - 1) * 240 + i, // Estimado
          embedding
        }
      });

      // Reportar progreso
      await job.updateProgress({
        part: partNumber,
        chunk: i + 1,
        totalChunks: chunks.length
      });
    }

    // Marcar como completado
    await prisma.legalDocumentPart.update({
      where: {
        parentDocumentId_partNumber: {
          parentDocumentId,
          partNumber
        }
      },
      data: {
        isProcessed: true,
        processingStatus: 'completed'
      }
    });

    // Verificar si todas las partes están procesadas
    const allParts = await prisma.legalDocumentPart.findMany({
      where: { parentDocumentId }
    });

    const allProcessed = allParts.every(p => p.isProcessed);

    if (allProcessed) {
      // Consolidar contenido en documento padre
      const fullContent = allParts
        .sort((a, b) => a.partNumber - b.partNumber)
        .map(p => p.content)
        .join('\n\n');

      await prisma.legalDocument.update({
        where: { id: parentDocumentId },
        data: { content: fullContent }
      });
    }

    return { success: true, partNumber, chunksGenerated: chunks.length };
  } catch (error) {
    await prisma.legalDocumentPart.update({
      where: {
        parentDocumentId_partNumber: {
          parentDocumentId,
          partNumber
        }
      },
      data: {
        processingStatus: 'failed',
        metadata: { error: (error as Error).message }
      }
    });

    throw error;
  }
}

Paso 7: Frontend - Mostrar Progreso

// frontend/src/components/DocumentUploadProgress.tsx

export function DocumentUploadProgress({ documentId }: { documentId: string }) {
  const [parts, setParts] = useState<any[]>([]);

  useEffect(() => {
    const interval = setInterval(async () => {
      const response = await fetch(`/api/legal-documents/${documentId}/parts`);
      const data = await response.json();
      setParts(data.parts);

      // Si todas están completadas, detener polling
      if (data.parts.every((p: any) => p.isProcessed)) {
        clearInterval(interval);
      }
    }, 2000);

    return () => clearInterval(interval);
  }, [documentId]);

  return (
    <div>
      <h3>Procesando documento en {parts.length} partes</h3>
      {parts.map(part => (
        <div key={part.partNumber}>
          <span>Parte {part.partNumber}/{part.totalParts}</span>
          <span>Páginas {part.startPage}-{part.endPage}</span>
          <span>{part.processingStatus}</span>
          {part.isProcessed ? '✅' : '⏳'}
        </div>
      ))}
    </div>
  );
}

✅ Recomendaciones Finales

🎯 Plan de Acción Recomendado

  1. Corto Plazo (Inmediato):
    • Implementar Alternativa 1 (División Física) para documentos >50 MB
    • Mantener procesamiento actual para documentos <50 MB
    • Configurar límite automático: dividir si fileSize > 50 MB
  2. Mediano Plazo (1-2 meses):
    • Implementar Alternativa 4 (Jerarquía) para códigos estructurados
    • Crear UI para ver progreso de procesamiento por partes
    • Agregar opción de descarga por partes o completo
  3. Largo Plazo (3-6 meses):
    • Migrar a Render Starter (512 MB RAM, $7/mes)
    • Considerar Alternativa 5 (S3) si volumen crece significativamente
    • Implementar compresión inteligente de PDFs antes de almacenar

Mejores Prácticas

✅ DO (Hacer)

  • Validar tamaño de archivo antes de procesamiento
  • Mostrar estimación de tiempo de procesamiento al usuario
  • Implementar reintentos granulares por parte fallida
  • Cachear embeddings en Redis (24h TTL)
  • Registrar métricas de procesamiento (tiempo, RAM, costos)
  • Permitir cancelación de procesamiento en curso
  • Generar notificación cuando procesamiento complete

❌ DON'T (No Hacer)

  • No cargar archivos >100 MB completos en memoria
  • No procesar todas las partes en un solo job (timeout risk)
  • No eliminar partes inmediatamente después de procesamiento
  • No exponer URLs de partes individuales públicamente
  • No confiar solo en validación frontend (validar en backend)

Consideraciones de Seguridad

🔒 Seguridad

  • Validación de tipo: Verificar que el archivo sea realmente PDF (magic bytes)
  • Sanitización: Eliminar metadata potencialmente maliciosa del PDF
  • Rate limiting: Limitar uploads por usuario (ej: 5 documentos/hora)
  • Autorización: Solo admins pueden subir documentos legales globales
  • Encriptación: Considerar encriptar partes en reposo si contienen info sensible
  • Auditoría: Registrar en AuditLog todas las divisiones y procesamientos

Escalabilidad Futura

📈 Cuando Escalar

Considerar upgrade de plan si se cumplen estos indicadores:

  • Procesamiento de >100 documentos grandes/mes
  • Timeouts frecuentes (>10% de fallos)
  • Quejas de usuarios sobre lentitud
  • Crecimiento >50% mensual en uploads

Opciones de Escalamiento:

  1. Render Starter ($7/mes): 512 MB RAM, más rápido
  2. Render Standard ($25/mes): 2 GB RAM, auto-scaling
  3. Workers dedicados: Separar procesamiento de API
  4. CloudFront + S3: Para archivos estáticos
