🤖 Extracción Automática de Metadatos con IA

Tu servidor de carga ahora extrae metadatos legales automáticamente usando GPT-4

✨ ¿Qué hace esto?

Cuando cargas un PDF, el sistema ahora extrae automáticamente toda la información legal importante usando inteligencia artificial:

📅
Fechas

Publicación y reformas

📋
Tipo de Norma

Constitución, Ley, Código, etc.

🏛️
Jerarquía

Nivel legal del documento

📰
Registro Oficial

Número y tipo de publicación

📊 Campos que se Extraen Automáticamente

📋 Tipo de Norma (normType)
Constitución, Ley Orgánica, Ley Ordinaria, Código, Reglamento, Ordenanza, Resolución, etc.
🏛️ Jerarquía Legal (legalHierarchy)
CONSTITUCION, TRATADOS_INTERNACIONALES_DDHH, LEYES_ORGANICAS, LEYES_ORDINARIAS, CODIGOS_ORGANICOS, CODIGOS_ORDINARIOS, REGLAMENTOS, ORDENANZAS, RESOLUCIONES
📰 Tipo de Publicación (publicationType)
ORDINARIO, SUPLEMENTO, SEGUNDO_SUPLEMENTO, SUPLEMENTO_ESPECIAL, EDICION_CONSTITUCIONAL
🔢 Número de R.O. (publicationNumber)
Número del Registro Oficial (ej: "449", "298")
📅 Fecha de Publicación (publicationDate)
Fecha exacta de publicación en Registro Oficial
🔄 Fecha de Última Reforma (lastReformDate)
Fecha de la reforma más reciente (si existe)
📄 Estado del Documento (documentState)
ORIGINAL (sin reformas) o REFORMADO (tiene modificaciones)
🌍 Jurisdicción (jurisdiction)
NACIONAL, PROVINCIAL, MUNICIPAL, INTERNACIONAL
📆 Año (year)
Año de publicación del documento
🎯 Confianza de Extracción
Nivel de confianza de la IA (0.0 a 1.0)

🔍 ¿Cómo Funciona?

1. Extracción de Texto

El sistema lee las primeras 5 páginas del PDF (donde suele estar toda la información legal)

2. Análisis con GPT-4

GPT-4 analiza el texto y busca patrones típicos de documentos legales ecuatorianos:

REGISTRO OFICIAL Órgano del Gobierno del Ecuador Año II - Quito, Jueves 20 de Octubre del 2008 - Nº 449 CONSTITUCIÓN DE LA REPÚBLICA DEL ECUADOR ...

3. Extracción Estructurada

La IA retorna un JSON con todos los campos:

{ "normType": "CONSTITUTIONAL_NORM", "legalHierarchy": "CONSTITUCION", "publicationType": "ORDINARIO", "publicationNumber": "449", "publicationDate": "2008-10-20", "lastReformDate": null, "documentState": "ORIGINAL", "jurisdiction": "NACIONAL", "year": 2008, "extractionConfidence": 0.95 }

4. Guardado en Base de Datos

Todos los campos se guardan automáticamente en la tabla LegalDocument en tu base de datos PostgreSQL en Render

⚡ Ventajas

  • ✅ 100% Automático: No necesitas escribir nada manualmente
  • 🎯 Alta Precisión: GPT-4 está entrenado para identificar estos patrones
  • ⏱️ Rápido: Toma solo 2-3 segundos adicionales
  • 📊 Completo: Extrae TODOS los campos necesarios
  • 🔄 Confiable: Si falla, usa valores por defecto seguros
  • 💾 Persistente: Todo se guarda en Render PostgreSQL

🎬 Ejemplo de Uso

Cuando cargas la Constitución de Ecuador:

1. Cargas el PDF: "Constitucion-Ecuador-2008.pdf"
2. El sistema extrae automáticamente:
📋 Tipo: CONSTITUTIONAL_NORM
🏛️ Jerarquía: CONSTITUCION
📰 Tipo Publicación: ORDINARIO
🔢 R.O. Nº: 449
📅 Fecha: 20 de Octubre de 2008
📄 Estado: ORIGINAL
🌍 Jurisdicción: NACIONAL
📆 Año: 2008
3. Todo se guarda en la base de datos con estos metadatos y el contenido completo procesado

🛠️ Tecnología Utilizada

  • GPT-4: Modelo de lenguaje para análisis semántico
  • pdf.js-extract: Extracción de texto del PDF
  • JSON Structured Output: Garantiza respuesta en formato JSON válido
  • Temperature 0.1: Respuestas más consistentes y precisas
  • Fallback Automático: Si falla, usa valores por defecto
