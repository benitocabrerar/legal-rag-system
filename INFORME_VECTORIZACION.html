5Pipeline técnico de vectorización

Stack

  • Extracción: pdf-parse nativo. Fallback a Claude Vision (Anthropic) si < 500 chars (PDFs escaneados).
  • Clasificación: heurística por nombre de archivo + análisis del contenido.
  • Chunking: 1.000 caracteres con overlap de 200 (≈ 250 tokens/chunk).
  • Embeddings: OpenAI text-embedding-3-small, dimensión 1536, batches de 50.
  • Persistencia: Supabase Postgres con pgvector. Index HNSW (m=16, ef_construction=64).
  • Búsqueda: RPC search_legal_chunks con RRF (Reciprocal Rank Fusion) entre semantic search y keyword search en español (websearch_to_tsquery + ts_rank_cd).

Metadata persistida por documento

  • Estructurada (campos típados): norm_type, legal_hierarchy, publication_type, publication_number, publication_date, last_reform_date, document_state, jurisdiction, category
  • JSONB metadata: file (sha256, tamaño), extraction (pages, words, method), pdf_native (Title/Author/Subject/Creator/Producer del PDF), pdf_xmp (metadata XMP), classification, detected (estructura: artículos/libros/títulos/capítulos, keywords, sample articles, references_to)
  • Chunks: id, legal_document_id, content, chunk_index, embedding_v vector(1536)

6Conclusiones

127 documentos vectorizados con éxito. 32.476 chunks indexados con embeddings de 1536 dimensiones. 27.742 artículos detectados automáticamente. Costo total estimado: $0.138 USD (6.91M tokens).

Lo que se hizo

  • Cada PDF clasificado en su nivel jerárquico según el Art. 425 CRE
  • Metadata nativa del PDF preservada (Title, Author, Producer, fechas, etc.)
  • Detección heurística de número de Registro Oficial, fecha de publicación y última reforma
  • Conteo automático de artículos, libros, títulos, capítulos y secciones
  • Lista de keywords jurídicas y referencias cruzadas a otras normas
  • Hash SHA-256 del archivo original para auditoría
  • Idempotente: re-runs detectan documentos ya vectorizados y los saltan

Próximos pasos sugeridos

  • Búsqueda federada: query única que combine corpus jurídico + casos del usuario
